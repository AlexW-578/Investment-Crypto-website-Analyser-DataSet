Transparency & Consent Framework (TCF 2.2) Compliance

Ads Interactive is fully compliant with the IAB Europe Transparency & Consent Framework (TCF) version 2.2. Our platform ensures that all ad request parameters pass user consent verification in accordance with GDPR regulations.

As part of the TCF 2.2 framework, we handle user consent and process bid requests based on the following principles:

Consent Validation & Data Processing Under TCF 2.2

1. GDPR Compliance - Flag (gdpr Parameter)

  • A static parameter indicating whether GDPR applies to the ad request.
  • If set to 1, the system validates user consent using the gdpr_consent string.
  • If missing, the system determines GDPR applicability based on the user’s IP address lookup.

Consent String (gdpr_consent Parameter)

  • A Base64-encoded TCF string containing user consent preferences.
  • If present, the system verifies that the consent is valid and matches the vendor’s registered purposes.
  • If missing or invalid, personal data is automatically anonymized before sending requests to demand-side partners (DSPs).

Data Anonymization in Case of Missing Consent

If a user does not provide valid consent, or if the consent string is missing, the following data is automatically anonymized or removed from the bid request:

Anonymized data

  • device.ip → The system masks the last octet of the IP address (e.g., 46.211.248.31 → 46.211.248.0).
  • device.geo & user.geo → Only country, region, and city are retained, while more precise location data is removed.

Removed data fields

  • user.id (User ID)
  • device.ifa
        (Identifier for Advertisers)
  • device.ipv6
  • device.didsha1, device.didmd5, device.dpidsha1, device.dpidmd5, device.macsha1, device.macmd5
  • user.yob (Year of Birth)
  • user.gender
  • user.keywords, user.data

This ensures that Ads Interactive does not process personal data without explicit user consent, in compliance with GDPR requirements.

Consent Verification & Vendor List Validation

  • The Ads Interactive system checks vendor registration every hour against the IAB Global Vendor List (GVL) at https://iabeurope.eu/vendor-list-tcf.
  • Only registered vendors in the GVL with the appropriate TCF 2.2 permissions are authorized to process user data.
  • If a vendor loses its authorization, the system automatically anonymizes user data in all ad requests associated with that vendor.

Handling of Direct and Reseller Inventory

  • Direct inventory: Publishers must enable GDPR compliance by switching on the GDPR toggle when creating/editing an inventory.
  • Reseller traffic: If an ad request originates from a regulated country and lacks a valid TCF 2.2 consent string, user data is automatically anonymized before being passed to downstream DSPs.

FAQs – Common Consent Scenarios

What happens if the publisher provides a valid TCF 2.2 string?

  • The consent string is validated, and data is processed only according to user consent preferences.

What happens if the system is not listed as an authorized vendor?

  • The request is forwarded without user data, ensuring compliance.

What happens if the user explicitly denies consent for data processing?

  • The consent string remains encoded, but all personal data fields are set to anonymized values before being passed to demand partners.

This structured approach ensures full GDPR and TCF 2.2 compliance, while maximizing transparency and user control over data processing.
