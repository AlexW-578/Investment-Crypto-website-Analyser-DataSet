High Level API, No DOM Hacking

Using the InboxSDK means no more lost time hacking Gmail's DOM.

No more finding intricate selectors to insert your own UI. No more testing every configuration of Gmail to make sure everything still works. No more guessing how the compose area will appear. No more late nights when Gmail slightly changes their DOM.

Instead, with InboxSDK, you interact with simple high level classes like ComposeView. You call straightforward methods (like adding buttons or getting the subject) on the class and the SDK abstracts the entire DOM away from you.

Performant By Default

Gmail and Inbox are highly complex single page web apps. Typical browser extensions slow down these apps if they aren't carefully made. The InboxSDK was designed to be performant while supporting multiple applications.

See the documentation on tips for keeping your apps fast and lean (the key to keeping your users happy and your app installed).
